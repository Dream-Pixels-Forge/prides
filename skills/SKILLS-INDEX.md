## Skill Creation Guidelines

### Structure

```
skills/{skill-name}/
├── SKILL.md              # Main skill definition (required)
├── LICENSE.txt           # License terms
├── resources/            # Optional resources
│   ├── getting-started.md
│   ├── quick-reference.md
│   └── examples/
├── examples/             # Optional examples
│   └── example1.md
└── agents/               # Optional specialized agents
    └── agent-name.md
```

### SKILL.md Frontmatter

```yaml
---
name: skill-name
description: Clear description of when and how to use
license: Complete terms in LICENSE.txt
---
```

### Best Practices

1. **Clear Naming:** Use descriptive, unique names
2. **Comprehensive Docs:** Include examples and use cases
3. **Resource Files:** Add quick references and guides
4. **Examples:** Provide working examples
5. **License:** Include proper licensing
